Présentation de CSS

  1. Structuration/organisation du contenu (texte, images, vidéos) : HTML
  2. Présentation (visuelle) du contenu : CSS
  3. Rendre ce contenu interactif : JavaScript

Présentation de CSS

  • Cascading StyleSheets
  • Ce n’est pas un langage de programmation
  • Il sert à décrire la présentation d'un document HTML
  • L’intérêt principal est la ré-utilisabilité, i.e. pouvoir appliquer un même style à plusieurs pages HTML

Démonstrations

Principe

  • Définir des règles de style qui s'appliquent aux éléments HTML
  • Ces règles de style sont définies sous la forme:
    • 
selecteur {
    propriété: valeur;
}
    où
    • selecteur est une expression qui permet de désigner un ou plusieurs éléments HTML sur lesquels va s’appliquer la règle de style
    • propriété est l'attribut CSS que l'on souhaite modifier (ex: color, font-size, etc.)
    • valeur est la valeur que l'on souhaite donner à la propriété (ex: red, 16px, etc.)

Principe

Exemple


html, body {
    margin: 0;
    padding: 0;
}
body {
    background-color: white;
    font-family: Verdana, sans-serif;
    font-size: 100%;
}
h1 {
    font-size: 200%;
    color: navy;
    text-align: center;
}
h2 {
    font-size: 150%;
    color: red;
    padding-left: 15px;
}

3 façons de définir des règles CSS

  1. dans les balises du fichier source HTML, via un attribut style
    2. 
<span style="color:red"> texte en rouge </span>
<p style="color:blue;margin-left:30px;"> paragraphe en bleu, avec une marge gauche </p>
  • Ne s'applique qu'à un seul élément HTML
  • On ne donne que les pairs propriété:valeur (pas besoin de sélecteur)
  • mélange structuration (HTML) et présentation (CSS)
  • Non recommandé (voire à proscrire !)

3 façons de définir des règles CSS

  1. dans l’entête du fichier HTML, avec les balises <style> </style>
    2. 
<head>
    <meta charset="utf-8">
    <title>Exemple</title>
    <style>
        p {
            color: blue;
            margin-left: 30px;
        }
    </style>
</head>
  • Meilleure séparation structuration / présentation
  • Un style pour un ou plusieurs éléments HTML
  • Utilisable pour un seul document HTML

3 façons de définir des règles CSS

  1. dans un fichier séparé, d’extension .css, rattaché à la page HTML
    2. 
<head>
    <meta charset="utf-8">
    <title>Exemple</title>
    <link rel="stylesheet" href="exemple.css" type="text/css">
</head>
  • Parfaite séparation structuration / présentation
  • Un style pour un ou plusieurs éléments HTML
  • Possibilité de réutiliser le style pour plusieurs pages HTML
  • Possibilité d’ajouter plusieurs feuilles de styles à une même page (les styles s’ajoutent)

5 types de CSS s'appliquent à une page (par ordre de priorité)

  1. styles définis par l’utilisateur : certains navigateurs permettent à l’utilisateur de définir des feuilles de styles CSS à appliquer à toutes les pages visitées.
  2. styles définis via les attributs style dans les balises
  3. styles définis dans l’entête du fichier HTML, avec les balises <style> </style>
  4. styles définis dans un fichier séparé, d’extension .css, rattaché à la page HTML
  5. styles définis par défaut par le navigateur : chaque navigateur applique des styles CSS par défaut (généralement simples)

Note: l'ordre de priorité est important, car il détermine quel style sera appliqué en cas de conflit. les styles 3. et 4. ont la même priorité et s'appliquent dans l'ordre de leur apparition dans le document HTML.

Les sélecteurs avancés

Sélecteurs avancés

  • Sélecteurs qui s'appuient sur la structure du document
  • CSS fournit des opérateurs pour sélectionner des éléments de cette structure
  • Syntaxe
    • 
selecteur1 operateur selecteur2 {
    ...
}
    où operateur est (space), >, + ou ~.

Descendant selectors

  • (space) : balises inclues dans d’autres balises
    • 
article h2 {...}
                    
    
<h2> Ce titre ne sera pas selectionne </h2>
<article>
    <h2> Ce titre sera selectionne </h2>
    <div>
        <h2> Ce titre sera selectionne </h2>
    </div>
</article>
    Peu importe le degré de descendance, toutes les balises h2 inclues dans une balise article sont concernées

Direct child

  • > : descendance directe
    • 
article > h2 {...}
                    
    
<h2> Ce titre ne sera pas selectionne </h2>
<article>
    <h2> Ce titre sera selectionne </h2>
    <div>
        <h2> Ce titre sera selectionne </h2>
    </div>
</article>
    Cette fois, le dernier h2 n’est plus sélectionné (pas fils, mais petit-fils de article)

Sibling

  • ~ : tous les éléments frères qui suivent
    • 
h2 ~ p {...}
                    
    
<p> Ce paragraphe n'est pas selectionne </p>
<section>
    <h2> Ce titre n'est pas selectionne </h2>
    <p> Ce paragraphe est selectionne </p>
    <div>
        <p> Ce paragraphe n'est pas selectionne </p>
    </div>
    <p> Ce paragraphe est selectionne </p>
</section>
    Le style porte sur le deuxième opérande (ici p)

Adjacent Sibling selectors

  • + : le premier élément frère adjacent
    • 
h2 + p {...}
                    
    
<p> Ce paragraphe n'est pas selectionne </p>
<section>
    <h2> Ce titre n'est pas selectionne </h2>
    <p> Ce paragraphe est selectionne </p>
    <div>
        <p> Ce paragraphe n'est pas selectionne </p>
    </div>
    <p> Ce paragraphe est selectionne </p>
</section>
    Même principe, mais cette fois les balises doivent ”être collées” l’une à l’autre (et dans l’ordre)

Pseudo-classes et pseudo-éléments

  • Link Pseudo-class : liens visités ou non
    • 
a:link {...}
a:visited {...}
  • User Action Pseudo-class : action utilisateur (pas que pour les liens)
    • 
a:hover {...}
button:focus {...}
  • Structural & Position Pseudo-class : position dans la structure/hiérarchie HTML
    • 
li:first-child {...}
div:only-child {...}
  • Pseudo-element : première lettre/ligne, sélection, ou insérer un contenu avant/après un élément
    • 
p::first-letter {...}
p::first-line {...}
p::selection {...}
Cascade, spécificité et héritage

Cascade

  • Les règles de style sont définies les unes en dessous des autres
  • Elles sont interprétées (appliquées) dans l'ordre où elles apparaissent
  • Permet d’ajouter des styles qui se complètent ou se remplacent les uns les autres
  • Exemple:
    • 
.surround { /* ajoute une bordure a un block */
    border: 1px solid gray;
    border-radius: 25px;
    margin: 10px 40px;
    padding: 10px;
}
.info { /* couleur de fond pour les messages d'informations */
    background-color: #DDEEFF;
}
                    
    
<p class="surround info">Voici un message d'information.</p>
  • Si conflits entre deux règles, la dernière est celle qui prévaut

Spécificités

  • Cette règle s’applique uniquement si le conflit ne peut pas se régler avec les spécificités
  • Les sélecteurs ne sont pas tous d'égale importance et certains sont prioritaires sur d’autres
  • Une spécificité est calculée pour déterminer l'importance d'un sélecteur
  • Elle est composée de 3 nombres :
    • a, le nombre d’identifiants présents dans le sélecteur
    • b, le nombre de classes, d’attributs ou de pseudo-classes
    • c, le nombre d’éléments ou de pseudo-éléments
  • On note la spécificité d'un sélecteur sous la forme a-b-c

Spécificités

Exemples (dans l'ordre inverse des priorités)

Sélecteur Spécificité Explications
div 0-0-1 Un élément
div span 0-0-2 Deux élément
div > ul::before 0-0-3 Deux éléments et un pseudo-élément
.info 0-1-0 Une classe
a[target=_blank] 0-1-1 Un élément et un attribut
div .champ[disabled] 0-2-1 Un élément, une classe et un attribut
.c1.c2.c3....c10 0-10-0 10 classes
#intro 1-0-0 Un identifiant
#main .inner:empty 1-2-0 Un identifiant, une classe et une pseudo-classe

Spécificités

  • Les spécificités priment sur la cascade
  • Pour cette raison, il est préférable de privilégier les spécificités les plus basses
  • Par exemple, en privilégiant les classes plutôt que les identifiants:
    • 
.btn {
    border: 1px solid black;
}
.btn-danger {
    background: red;
}
.btn-success {
    background: green;
}
                    
    
<a class="btn btn-danger"> ... </a>
<a class="btn btn-success"> ... </a>

Héritage

  • À ces deux principes, s'ajoute l'héritage des styles
  • La structure d'un document HTML, implique que des éléments ont
    1. des ancêtres
    2. un parent
    3. des frères
    4. des enfants
    5. des descendants.
  • Un style appliqué à un élément est appliqué à ses descendants, sauf si une autre règle annule cet héritage
  • Toutes les propriétés ne sont pas concernées
  • Pour vérifier, si une propriété est héritée par défaut :https://www.w3.org/TR/CSS/#properties

Note: Le sélecteur universel * empêche l’héritage car il définit des propriétés pour tous les éléments HTML. Donc à utiliser avec prudence/parcimonie

Mise en page CSS

Modèle en boîte

Pour CSS, tous les éléments d'une page HTML sont des boîtes rectangulaires définies par :

  1. une largeur et une hauteur : width et height
  2. des espacements intérieurs : padding
  3. des espacements extérieurs : margin
  4. une bordure : border

Modèle en boîte


div {
    border: 6px solid #949599;
    width: 400px;
    height: 100px;
    margin: 20px;
    padding: 20px;
}
  • Largeur : 492px = 20px + 6px + 20px + 400px + 20px + 6px + 20px
  • Hauteur : 192px = 20px + 6px + 20px + 100px + 20px + 6px + 20px

Display

  • Les éléments HTML est par défaut de type "bloc" ou "en ligne" (block ou inline)
  • CSS permet de modifier ce type à l’aide de la propriété display
    • 
span {
    display: inline;
}
  • Rappel:
    • block : occupe toute la largeur + retours à la ligne avant/après
    • inline : ne prend que la place nécessaire + pas de retours à la ligne
  • Autres valeurs possible:
    • none : l’élément n’est pas affiché (mais existe toujours dans l'arbre)
    • inline-block : idem que block mais sans retours à la ligne

Display

Exemple


span.a {
    display: inline;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid blue;
    background-color: yellow;
}
span.b {
    display: inline-block;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid blue;
    background-color: yellow;
}
span.c {
    display: block;
    width: 100px;
    height: 100px;
    padding: 5px;
    border: 1px solid blue;
    background-color: yellow;
}

Positionnement

  • Positionnement des éléments avec la propriété position
  • Valeurs possibles :
    • static : valeur par défaut, positionnement normal
    • relative : positionnement relatif à sa position d'origine
    • absolute : positionnement par rapport à son ancêtre le plus proche ayant une position autre que static
    • fixed : positionnement par rapport à la fenêtre du navigateur
    • sticky : positionnement hybride, reste fixe après avoir atteint un certain défilement
  • Autre positionement: flottant à droite ou à gauche dans son conteneur
    • 
.float {
    float: left; /* ou right */
    width: 100px;
    margin: 10px;
}

Remarque sur le positionnement flottant

  • Les positionnements flottants sont (étaient?) très utilisés les mises en page en colonnes
  • Mais posent plusieurs difficultés:
    • Les éléments flottants peuvent provoquer des chevauchements inattendus.
    • Le conteneur parent peut ne pas s'ajuster à la hauteur des éléments flottants.
  • Anciennes solutions: propriétés clear, overflow, etc. (cf. cet article)
  • Nouvelles solutions: flexbox, grid (que nous verrons au prochain chapitre)
Conseils et bonnes pratiques

Conseils et bonnes pratiques

  • Proscrire les attributs style dans les balises HTML
  • Ne pas combiner balises style et fichiers .css car les priorités sont souvent difficiles à anticiper
  • Définir les styles du plus générique au plus spécifique
  • Privilégier les class aux id et utiliser ces derniers à la fin
  • Définir des styles modulaires plutôt que globaux
  • Utiliser les resets CSS (détails en TP)
  • Utilise l'inspecter du navigateur le plus possible pour analyser le résultat
