HTML

Préparation

Pour ces TP, vous allez principalement concevoir des pages HTML que vous allez enregistrer dans un dossier personnel et que vous allez tester dans un navigateur web. Vous pouvez utiliser l'éditeur de texte ou l'IDE de votre choix pour éditer ces pages. En revanche, il est préférable d'utiliser le navigateur Mozilla Firefox pour les tests. Si vous ne souhaitez pas utiliser Firefox, un autre navigateur fera tout à fait l'affaire mais les manipulations décrites dans cet énoncé devront être adaptées.

Pour la mise en place de vos outils :

  • Créez un répertoire de travail dans votre dossier personnel (Z:), dans lequel vous placerez vos fichiers sources.
  • Créez également un premier sous-dossier pour cet exercice. Vous ferez de même pour chaque exercice.
  • Créez et éditez un fichier HTML avec le code source ci-dessous.
    •

        <title>Premier exercice</title>
    </head>

            Web</em> du Master SID de l'Université de Rouen Normandie.

  • Ouvrez ce fichier dans un navigateur web.
  • Examinez cette page avec l'outil inspecter de Firefox, accessible dans les outils pour le développement ou via un "clic droit" sur la page.
  • Avec ce même outil, examinez le rendu de cette page en vue adaptative, c'est-à-dire comme si elle était affichée sur un terminal mobile.

Validation

Détecter des erreurs de syntaxe ou de structure dans un fichier HTML est compliqué car il n'y a pas de compilation. À l'interprétation, les navigateurs analysent le langage de façon très permissive et peuvent même parfois apporter des corrections automatiques pour réaliser l'affichage.

  • Téléchargez le fichier debug-example.html mis à disposition sur UniversiTICE.
  • Ouvrez-le dans un navigateur et ouvrez l'outil inspecter.
  • Le code HTML qui apparaît dans la fenêtre du bas n'est pas exactement identique au code du fichier téléchargé. Le navigateur a opéré à des corrections automatiques : lesquelles ?
  • Ouvrez maintenant le validateur du W3C : https://validator.w3.org/.
  • Faites le diagnostique du fichier debug-example.html.
  • En vous aidant du compte-rendu, corrigez toutes les erreurs jusqu'à ce que le validateur n'en détecte plus aucune.

  4. La vérification de cette page HTML (https://validator.w3.org/) ne doit retourner aucune erreur ni avertissement.

Préparation

Mise en place

Vous avez vu en cours qu'il y avait trois façons d'appliquer des styles CSS à une page HTML :

  • en utilisant l'attribut style dans les balises HTML
  • en utilisant la balise style dans l'entête du fichier HTML
  • en définissant les styles dans un fichier d'extension .css et en l'associant à la page HTML

C'est la troisième méthode que vous devez utiliser dans ce TP :

  1. Dans un nouveau dossier, faites une copie de la page HTML créée à l'exercice 1.3.
  2. Dans le même dossier, créez un nouveau fichier d'extension .css, et définissez-y une première règle de style pour changer la police de caractères à Gill Sans pour toute la page. Si cette police n'est pas prise en charge par le navigateur, il devra utiliser la police générique sans-serif.
  3. Appliquez cette feuille de style à la page HTML et visualisez le résultat dans un navigateur.
  4. Modifiez la feuille de style pour changer la couleur de fond de la page. Choisissez la couleur de votre choix (https://www.w3schools.com/colors/colors_names.asp).

Variables CSS

La prochaine étape consiste à établir la charte graphique. La plupart du temps, c'est un professionnel du Web design qui se charge de cette étape car cela demande surtout des compétences de graphiste. Pour ce TP, vous allez vous contenter de définir les polices de caractères et les couleurs de la page.

Pour cela, nous proposons d'utiliser des variables CSS comme montré dans l'exemple suivant :


:root {
    --background-color: #F5F5F6;
}
body {
    background-color: var(--background-color);
}

Le nom d'une variable CSS doit commencer par --. Elle peut ensuite être réutilisée dans la feuille de style à l'aide de la fonction var(). La portée d'une variable correspond à l'élément HTML désigné par le sélecteur de la règle dans laquelle elle est déclaré. Ici, nous utilisons la pseudo-classe :root qui permet de désigner l'élement racine de la page, afin que la variable soit accessible pour l'ensemble des règles de style qu'elle contient.

Vous allez donc définir 5 variables correspondant à 5 couleurs différentes :

  1. déclarez deux variables pour les couleurs d'arrière-plan, nommées --background-color1 et --background-color2
  2. déclarez trois variables pour les couleurs d'avant-plan, nommées --foreground-color1, --foreground-color2 et --foreground-color3
  3. utilisez ces variables pour changer la couleur d'arrière-plan de votre page et la couleur du texte.

N'hésitez pas à chercher sur internet des jeux de couleurs harmonieuses (e.g. avec https://coolors.co/). Si le sujet du choix judicieux des couleurs vous intéresse, vous pouvez lire cet article.

Réparer la navigation

En testant votre page sur différents navigateurs, vous devez constater que les liens ne sont pas très pratiques :

  • il n'existe pas de moyen de revenir en haut de la page sans avoir à "scroller" toute la page.
  • le haut de l'article que chaque lien est sensé afficher est caché par le menu de navigation.

Voici des éléments de solutions pour ces deux problèmes :

  • Pour le premier problème, une solution courante est d'ajouter un lien sur le logo de la barre de navigation, qui dirige vers le haut de la page.
  • Pour le deuxième problème, il existe plusieurs solutions documentées sur Internet. L'une d'elles consiste à définir la cible du lien avec une balise div vide, placée juste avant la cible réelle. Ensuite, avec CSS, il faut modifier le positionnement relatif de la cible pour le déplacer vers le haut, à l'aide de la pseudo-classe :target.

Bootstrap

Présentation de Bootstrap

Bootstrap est un framework de développement web front-end, c’est-à-dire dédié à la conception web coté client. Les caractéristiques notables de ce framework sont:

  • Open source et gratuit (licence MIT).
  • Très bien documenté et porté par une communauté active.
  • Basé sur un agencement en grille de 12 colonnes, s'appuyant sur des Flexbox CSS.
  • Utilise Normalize.css en remplacement des resets CSS, jugés parfois trop simplistes.
  • Compatible avec des outils de pré-compilation CSS, comme SASS.

La dernière version finalisée de Bootstrap est la version 5 (mai 2021). Elle intègre les standards de l’open web platform les plus récents. Pour cette raison, il est conseillé d’utiliser cette version. À noter que les versions antérieures s’appuient sur le framework Javascript JQuery, qu’il faut installer indépendamment. Cette dépendance a été supprimée à partir de la version 5.

Bootstrap se présente sous la forme de fichiers CSS et Javascript, produits par des outils de génération de code CSS/Javascript dont le but est de produire des fichiers prêts à l’emploi et optimisés pour un chargement aussi rapide que possible.

Les fichiers CSS (.css) quant à eux, sont fournis dans plusieurs versions:

  • avec le suffixe .min: versions minifiées, c'est-à-dire épurées des commentaires, espaces, sauts de lignes, etc.
  • avec le suffixe -grid: versions contenant le système de grille uniquement.
  • avec le suffixe -rtl: versions pour les langues s'écrivant de droite à gauche.
  • avec le suffixe -reboot: versions contenant la réinitialisation des styles par défaut uniquement.
  • avec le suffixe -utilities: versions contenant les utilitaires de Bootstrap uniquement.

Un seul de ces fichiers est à ajouter aux pages HTML. Typiquement, bootstrap.min.css permet de bénéficier de toutes les fonctionnalités CSS de Bootstrap. Certaines fonctionnalités avancées auront besoin du fichier bootstrap.min.js pour fonctionner.

Installation

Pour utiliser Bootstrap, vous pouvez télécharger les fichiers présentés ci-dessus et de les copier dans votre application web. Il faut ensuite les inclure dans les fichiers HTML comme ceci :


<!doctype html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <link href="css/bootstrap.min.css" rel="stylesheet">
    </head>
    <body>
        ...
        <script src="js/bootstrap.min.js"></script>
    </body>
</html>

Une autre possibilité est d'utiliser un CDN (Content Delivery Network) pour éviter de télécharger et copier les fichiers dans chacun de vos projets. Pour plus de détails, consultez la documentation de Bootstrap.

Grille flexible

Le système de grille flexible de Bootstrap s'appuit sur un élément conteneur qui encapsule le contenu de la grille. Ce conteneur permet d'appliquer les comportements responsives.

Ces comportements s'appuient sur la définition de 6 points de ruptures (breakpoints):

  • <576px: xs, écrans très petits (smartphone en mode portrait)
  • ≥576px: sm, écrans petits (smartphone en mode paysage)
  • ≥768px: md, écrans moyens (tablette en mode portrait)
  • ≥992px: lg, écrans larges (tablette en mode paysage)
  • ≥1200px: xl, écrans très larges (ordinateur de bureau)
  • ≥1400px: xxl, écrans très très larges (grand écran de bureau)

Pour définir un élément conteneur, il faut affecter l'une des classes CSS suivantes à un élément HTML:

  • .container: ajoute des marges automatiques sur les côtés
  • .container-fluid: prend toute la largeur de l'écran, quel qu'il soit.
  • .container-{breakpoint}: occupe toute la largeur de l'écran jusqu'au point de rupture spécifié. {breakpoint} est à remplacer par l'un des 5 suffixes: sm, md, lg, xl, xxl.

Par exemple:


<main class="container-md">
    ...
</main>

Dans cet exemple, l'élément main occupe toute la largeur de son conteneur tant que cette largeur est inférieure à 768px. Au delà, des espacements seront ajoutés de chaque côté.

La grille de Bootstrap est composé de 12 colonnes de largeur identique. Pour y placer des éléments HTML, il faut:

  • définir une ligne avec un élément HTML auquel on affecte la classe .row
  • définir des cellules à l'intérieur de cette ligne, avec des éléments HTML auxquels on affecte la classe .col-{n} où {n} est un entier compris entre 1 et 12 permettant de spécifier la largeur en nombre de colonnes.

Bootstrap décline les classes .col-{n} en 5 variantes qui permettent d’ajouter une contrainte sur le comportement responsive des éléments: col-sm-x, col-md-x, col-lg-x, col-xl-x et col-xxl-x. Le principe de ces déclinaisons est d’autoriser les éléments à se ré-agencer en colonne plutôt qu’en ligne quand l’écran devient trop petit.

Par exemple:


<div class="container-md">
    <div class="row">
        <div class="col-md-4">Colonne 1</div>
        <div class="col-md-4">Colonne 2</div>
        <div class="col-md-4">Colonne 3</div>
    </div>
</div>

Dans cet exemple, les trois colonnes sont affichées côte à côte lorsque la largeur de l'écran est supérieure ou égale à 768px (point de rupture md). En dessous de cette largeur, les colonnes sont affichées les unes en dessous des autres.

Pour plus de détails, consultez la documentation de Bootstrap sur la grille.

Il est possible d'ajouter des espacements entre les colonnes et les lignes de la grille, appelés gutter. Bootstrap propose également des classes pour faire cela. Consultez la documentation pour savoir comment utiliser ces classes.

Barre de navigation

Les barres de navigation sont complexes à mettre en place, surtout lorsqu’il s’agit de définir un comportement responsive. Bootstrap fournit des outils puissants pour définir des barres de navigation qui s’adaptent automatiquement en fonction du terminal, y compris dans l’ergonomie.

L’élément central est la classe navbar. On l’applique le plus souvent à une balise nav. Cette classe définit un comportement par défaut complètement responsive. Beaucoup de classes additionnelles permettent de personnaliser ce comportement.

Dans une barre de navigation, on trouve souvent deux types d’éléments:

  • un logo/titre (en anglais brand), défini à l’aide de la classe navbar-brand.
  • un menu de navigation sous la forme d’une liste de liens, défini à l’aide de la classe navbar-nav.

Par exemple, voici un squelette type pour une barre de navigation Bootstrap :


<nav class="navbar">
    <a class="navbar-brand" href="#">
        <img src="img/logo.jpg" alt="logo">
    </a>
    <ul class="navbar-nav">
        <li class="nav-item active">
            <a class="nav-link" href="#">Menu 1</a>
        </li>
        <li class="nav-item">
            <a class="nav-link" href="#">Menu 2</a>
        </li>
        ...
    </ul>
</nav>

Les classes nav-item et nav-link permettent de définir les items du menu. Des styles, espacements et positionnement par défaut leur seront appliqués, en respectant les préceptes de la conception responsive.

Le comportement traditionnel d'une barre de navigation pour les terminaux mobiles est de cacher le menu à l’aide d’un bouton "tiroir". Vous pouvez trouvez un exemple de ce fonctionnement sur le site du Master (mastersd.univ-rouen.fr).

Ce mécanisme est facile à mettre en œuvre avec Bootstrap à l’aide d’un bouton à bascule (toggle) et de classes spéciales associées au menu. Il faut:

  1. Encapsuler le menu dans un élément div auquel on associe les classes collapse et navbar-collapse.
  2. Ajouter un bouton avant le menu, avec la classe navbar-toggler et l’attribut data-toggle="collapse".
  3. Ajouter au div précédent un id dont la valeur doit également être associée à l’attribut data-target du bouton.

Par ailleurs, il est possible de préciser à partir de quel point de rupture on souhaite que les menus soient ré-agencés verticalement (avec ou sans bouton à bascule). Il suffit pour cela d’associer la classe navbar-expand-* à l’élément navbar, en remplaçant * par xl, lg, md ou sm.
